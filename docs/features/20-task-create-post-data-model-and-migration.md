## What it does\nDefines the core Post data model for the blog, including SQLAlchemy and Pydantic schemas, and applies the initial Alembic migration to create the posts table. This enables all future API and data work related to blog posts.\n\n## How to use it\nN/A — no public interface\n\n## Internals\n- models.py: Adds the Post SQLAlchemy model with fields: id, title, content, author, status, created_at, updated_at. Defines the PostStatus enum for allowed status values.\n- alembic/versions/20260315101949_create_post_table.py: Alembic migration script to create the posts table with all required fields and enum constraints.\n- main.py: Imports and registers the Post model to ensure metadata is available for migrations and app startup.\n- schemas.py: Defines Pydantic schemas PostCreate, PostUpdate, and PostResponse for data validation and serialization.\n- tests/test_models.py: Unit tests for Post model field validation, required/optional fields, status values, and date handling.\n
